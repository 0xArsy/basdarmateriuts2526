SECTION 01

Conceptual Deep Dive

Apa itu SQL?

SQL (Structured Query Language, diucapkan /ˈsiːkwəl/) adalah bahasa standar untuk berinteraksi dengan basis data relasional. SQL bersifat declarative — kita menyatakan apa yang diinginkan, bukan bagaimana cara mendapatkannya.

STANDARDIZED

Banyak fitur baru seiring waktu, namun tetap kompatibel lintas DBMS.

INTERACTIVE

Via GUI, command prompt, atau di-embed dalam program.

DECLARATIVE

Berbasis relational algebra — deskripsikan hasil, bukan langkah.

DDL vs DML — Mental Model

DDL — Data Definition Language

Mendefinisikan struktur basis data. Ibarat "membangun gedung" — membuat denah, dinding, pintu.

CREATE TABLE ALTER TABLE DROP TABLE CREATE SCHEMA
DML — Data Manipulation Language

Memanipulasi isi data di dalam struktur. Ibarat "mengelola isi gedung" — memasukkan, memindahkan, membuang perabot.

INSERT UPDATE DELETE SELECT
KEY TERM

Schema

Wadah yang mengelompokkan tabel, view, constraint, dan elemen lain yang terkait dalam satu basis data. Diidentifikasi dengan nama schema. Dibuat dengan CREATE SCHEMA.

KEY TERM

Domain

Tipe data buatan pengguna yang membungkus tipe dasar. Memudahkan perubahan tipe sekaligus untuk banyak atribut. Contoh: CREATE DOMAIN SSN_TYPE AS CHAR(9).

KEY TERM

Base Relation vs Virtual Relation

Base: Dibuat dengan CREATE TABLE, data benar-benar tersimpan. Virtual: Dibuat dengan CREATE VIEW, tidak punya file fisik sendiri.

KEY TERM

Referential Integrity

Jaminan bahwa setiap nilai foreign key di satu tabel harus ada di tabel yang dirujuk. Ditegakkan otomatis oleh DBMS saat data dimodifikasi.

KEY TERM

ON DELETE / ON UPDATE Action

Tindakan saat data yang dirujuk FK dihapus/diubah: CASCADE (ikut hapus/ubah), SET NULL, SET DEFAULT, RESTRICT.

KEY TERM

Named Constraint

Constraint yang diberi nama dengan keyword CONSTRAINT nama. Memudahkan identifikasi error dan memungkinkan DROP/ALTER constraint secara spesifik.

⚠ Common Pitfalls
  • NOT NULL saat ALTER TABLE — Constraint NOT NULL tidak bisa ditambahkan langsung saat ALTER TABLE ADD COLUMN. Kolom baru selalu nullable karena semua baris lama tidak punya nilai.
  • Circular FK Reference — Tabel A punya FK ke B, dan B punya FK ke A. Solusi: buat tabel dulu tanpa FK, lalu tambahkan FK via ALTER TABLE.
  • Self-referencing FK — FK yang merujuk tabel itu sendiri (misal Super_ssn → Ssn di tabel EMPLOYEE) harus ditambahkan setelah tabel selesai dibuat.
  • DROP SCHEMA RESTRICT vs CASCADE — RESTRICT gagal jika schema tidak kosong. CASCADE menghapus SEMUA isinya. Berhati-hati menggunakan CASCADE di production!
  • INSERT tanpa nama kolom — Nilai harus persis sesuai urutan definisi kolom di CREATE TABLE. Jika ada kolom yang dilewati, gunakan form eksplisit dengan nama kolom.
  • INSERT ... SELECT tidak auto-update — Data yang di-insert dari query tidak otomatis tersinkronisasi saat sumber berubah. Gunakan VIEW untuk ini.
SECTION 02

DDL — Data Definition Language

Step-by-Step: Membangun Skema COMPANY

1

CREATE SCHEMA — Membuat Wadah Basis Data

Sebelum membuat tabel, buat schema sebagai kontainer logis. Berikan nama dan otorisasi pengguna.

SQL · CREATE SCHEMADDL
-- Membuat schema baru bernama COMPANY
-- Diotorisasi oleh user Jsmith
CREATE SCHEMA COMPANY AUTHORIZATION Jsmith;

-- DROP RESTRICT: gagal jika schema tidak kosong
DROP SCHEMA COMPANY RESTRICT;

-- DROP CASCADE: hapus semua isi schema
DROP SCHEMA COMPANY CASCADE; -- ⚠ hati-hati!

-- Membuat domain khusus (opsional)
CREATE DOMAIN SSN_TYPE AS CHAR(9);
-- Digunakan untuk: Ssn, Super_ssn, Mgr_ssn, Essn

💡 Domain sangat berguna: jika tipe CHAR(9) harus diubah ke VARCHAR(11), cukup ubah domain — semua atribut yang menggunakannya otomatis ikut berubah.

2

CREATE TABLE — Tabel EMPLOYEE & DEPARTMENT

Definisikan atribut, tipe data, dan constraint dasar. FK yang circular ditambahkan kemudian via ALTER TABLE.

SQL · EMPLOYEE + DEPARTMENTFigure 6.1
CREATE TABLE EMPLOYEE (
  Fname      VARCHAR(15)    NOT NULL,
  Minit      CHAR,
  Lname      VARCHAR(15)    NOT NULL,
  Ssn        CHAR(9)        NOT NULL,
  Bdate      DATE,
  Address    VARCHAR(30),
  Sex        CHAR,
  Salary     DECIMAL(10,2),
  Super_ssn  CHAR(9),       -- FK ditambahkan nanti (self-ref)
  Dno        INT            NOT NULL,
  PRIMARY KEY (Ssn)
);

CREATE TABLE DEPARTMENT (
  Dname          VARCHAR(15)  NOT NULL,
  Dnumber        INT          NOT NULL,
  Mgr_ssn        CHAR(9)      NOT NULL,
  Mgr_start_date DATE,
  PRIMARY KEY (Dnumber),
  UNIQUE (Dname),
  FOREIGN KEY (Mgr_ssn) REFERENCES EMPLOYEE(Ssn)
);
3

CREATE TABLE — Tabel Relasional Lanjutan

DEPT_LOCATIONS, PROJECT, WORKS_ON, DEPENDENT — masing-masing dengan composite PK dan multiple FK.

SQL · Tabel LanjutanFigure 6.1
CREATE TABLE DEPT_LOCATIONS (
  Dnumber   INT           NOT NULL,
  Dlocation VARCHAR(15)   NOT NULL,
  PRIMARY KEY (Dnumber, Dlocation),
  FOREIGN KEY (Dnumber) REFERENCES DEPARTMENT(Dnumber)
);

CREATE TABLE WORKS_ON (
  Essn  CHAR(9)       NOT NULL,
  Pno   INT           NOT NULL,
  Hours DECIMAL(3,1)  NOT NULL,
  PRIMARY KEY (Essn, Pno),
  FOREIGN KEY (Essn) REFERENCES EMPLOYEE(Ssn),
  FOREIGN KEY (Pno)  REFERENCES PROJECT(Pnumber)
);

CREATE TABLE DEPENDENT (
  Essn           CHAR(9)      NOT NULL,
  Dependent_name VARCHAR(15)  NOT NULL,
  Sex            CHAR,
  Bdate          DATE,
  Relationship   VARCHAR(8),
  PRIMARY KEY (Essn, Dependent_name),
  FOREIGN KEY (Essn) REFERENCES EMPLOYEE(Ssn)
);
4

ALTER TABLE — Modifikasi Struktur Tabel

Menambah kolom baru atau FK yang tidak bisa dibuat saat CREATE TABLE (circular reference, self-reference).

SQL · ALTER TABLEDDL
-- Menambah kolom baru (kolom baru = NULL di semua baris)
ALTER TABLE EMPLOYEE ADD JOB VARCHAR(12);
-- ⚠ NOT NULL tidak boleh di sini! Semua baris lama = NULL
-- Gunakan UPDATE setelah ini untuk mengisi nilai JOB

-- Menambah FK self-referencing (tidak bisa di CREATE TABLE)
ALTER TABLE EMPLOYEE
  ADD CONSTRAINT EMP_SUPER_FK
  FOREIGN KEY (Super_ssn) REFERENCES EMPLOYEE(Ssn);

-- Menambah FK dari EMPLOYEE ke DEPARTMENT
ALTER TABLE EMPLOYEE
  ADD CONSTRAINT EMP_DEPT_FK
  FOREIGN KEY (Dno) REFERENCES DEPARTMENT(Dnumber);

💡 Mengapa EMPLOYEE dan DEPARTMENT saling referensi? EMPLOYEE punya FK Dno → DEPARTMENT, sementara DEPARTMENT punya FK Mgr_ssn → EMPLOYEE. Karena circular, salah satunya harus ditambahkan via ALTER TABLE setelah keduanya terbuat.

5

DROP TABLE — Menghapus Tabel

Menghapus tabel beserta definisinya. Setelah di-DROP, tabel tidak bisa digunakan dalam query maupun command lain.

SQL · DROP TABLEDDL
-- Menghapus tabel DEPENDENT
DROP TABLE DEPENDENT;
-- Tabel + definisi hilang sepenuhnya
-- Tidak bisa di-query, di-update, atau dirujuk lagi

-- Jika ada tabel lain yang punya FK ke DEPENDENT,
-- DROP mungkin gagal (tergantung DBMS & setting CASCADE)

🚨 Perhatian: DROP TABLE tidak bisa di-undo! Pastikan tidak ada data penting yang akan hilang, dan pastikan tidak ada tabel lain yang punya FK ke tabel ini.

SECTION 03

Specifying Constraints in SQL

NOT NULL— Nilai wajib diisi

Kolom tidak boleh berisi NULL. Secara implisit sudah diterapkan untuk PRIMARY KEY.

CREATE TABLE PRODUCTS (
  Product_no INT         NOT NULL,
  Name       VARCHAR(10) NOT NULL,
  Price      INT  ← boleh NULL
);
UNIQUE— Nilai unik per kolom

Semua nilai di kolom harus berbeda satu sama lain. Berbeda dengan PK: UNIQUE boleh NULL (di beberapa DBMS, satu NULL diizinkan).

CREATE TABLE DEPARTMENT (
  Dnumber INT PRIMARY KEY,
  Dname   VARCHAR(15) UNIQUE
  -- Dua dept tidak boleh nama sama
);
PRIMARY KEY— Identitas unik baris

Kombinasi UNIQUE + NOT NULL. Satu tabel hanya boleh punya satu PK (bisa composite).

-- Single PK
CREATE TABLE PRODUCTS (
  Product_no INT PRIMARY KEY, ...
);
-- Composite PK
CREATE TABLE EXAMPLE (
  A INT, B INT, C INT,
  PRIMARY KEY (A, C)
);
CHECK— Kondisi kustom

Nilai harus memenuhi ekspresi boolean tertentu. Bisa diberi nama dengan CONSTRAINT untuk kemudahan identifikasi error.

CREATE TABLE PRODUCTS (
  Price INT CHECK (price > 0)
);
-- Dengan nama constraint:
Price INT CONSTRAINT POS_PRICE
  CHECK (price > 0)
FOREIGN KEY— Referential Integrity

Nilai FK harus ada di tabel rujukan. Bisa multi-kolom. Bisa diberi aksi ON DELETE / ON UPDATE.

FOREIGN KEY (B, C)
  REFERENCES OTHER(C1, C2)
  ON DELETE SET NULL
  ON UPDATE CASCADE;
DEFAULT— Nilai bawaan

Jika INSERT tidak menyertakan nilai untuk kolom ini, nilai default yang digunakan. Jika tidak ada DEFAULT, nilai otomatis NULL.

Price INT DEFAULT 9.99,
Dno   INT NOT NULL DEFAULT 1,
Mgr_ssn CHAR(9) NOT NULL
  DEFAULT '888665555'

SECTION 04

DML — INSERT, DELETE, UPDATE

Walkthrough: Memodifikasi Data

1

INSERT — Menambah Baris (Cara Dasar)

Nilai harus persis sesuai urutan kolom di CREATE TABLE. Semua kolom wajib diisi (atau berisi DEFAULT/NULL).

SQL · INSERT INTO ... VALUESU1
-- Cara 1: Urutan mengikuti definisi tabel (posisional)
INSERT INTO EMPLOYEE
VALUES (
  'Richard',     -- Fname
  'K',           -- Minit
  'Marini',      -- Lname
  '6532988653', -- Ssn
  '1962-12-30', -- Bdate
  '98 Oak Forest, Katy, TX', -- Address
  'M',           -- Sex
  37000,         -- Salary
  '653298653',  -- Super_ssn
  4              -- Dno ← urutan HARUS sama
);

-- Cara 2: Eksplisit nama kolom (lebih aman)
INSERT INTO EMPLOYEE (Fname, Lname, Ssn)
VALUES ('Richard', 'Marini', '653298653');
-- Kolom lain = NULL atau DEFAULT

💡 Best Practice: Selalu gunakan form eksplisit dengan nama kolom. Lebih aman, tidak terpengaruh jika urutan kolom berubah, dan lebih mudah dibaca.

2

INSERT dari Hasil Query (INSERT ... SELECT)

Memasukkan banyak baris sekaligus dari hasil SELECT. Data snapshot saat itu — tidak auto-update saat sumber berubah.

SQL · INSERT ... SELECTU3A + U3B
-- Buat tabel ringkasan departemen
CREATE TABLE DEPTS_INFO (
  DEPT_NAME   VARCHAR(10),
  NO_OF_EMPS  INTEGER,
  TOTAL_SAL   INTEGER
);

-- Isi dari hasil query agregasi
INSERT INTO DEPTS_INFO (DEPT_NAME, NO_OF_EMPS, TOTAL_SAL)
SELECT Dname, COUNT(*), SUM(Salary)
FROM DEPARTMENT, EMPLOYEE
WHERE Dnumber = Dno
GROUP BY Dname;

-- ⚠ Jika EMPLOYEE atau DEPARTMENT berubah setelah ini,
--   DEPTS_INFO TIDAK otomatis ikut berubah!
--   Gunakan CREATE VIEW untuk data yang selalu up-to-date.
3

DELETE — Menghapus Baris

Gunakan WHERE untuk memilih baris yang dihapus. Tanpa WHERE, SEMUA baris dihapus!

SQL · DELETE FROM ... WHEREU4A–U4D
-- U4A: Hapus berdasarkan nilai kolom
DELETE FROM EMPLOYEE WHERE LName = 'Brown';

-- U4B: Hapus berdasarkan primary key
DELETE FROM EMPLOYEE WHERE Ssn = '123456789';

-- U4C: Hapus berdasarkan foreign key
DELETE FROM EMPLOYEE WHERE Dno = 5;

-- U4D: Hapus SEMUA baris (tabel kosong, struktur tetap)
DELETE FROM EMPLOYEE;
-- ⚠ BERBEDA dengan DROP TABLE yang juga hapus struktur!

🚨 Ingat! DELETE FROM tabel (tanpa WHERE) menghapus semua data tapi tabel tetap ada. DROP TABLE tabel menghapus data sekaligus strukturnya.

4

UPDATE — Memodifikasi Nilai Kolom

Gunakan SET untuk menentukan kolom + nilai baru. WHERE memfilter baris yang diubah. Nilai baru bisa bergantung pada nilai lama.

SQL · UPDATE ... SET ... WHEREU5 + U6
-- U5: Update sederhana
UPDATE PROJECT
SET   Plocation = 'Bellaire',
      Dno       = 5
WHERE Pnumber = 10;

-- U6: Update dengan subquery (nilai baru tergantung nilai lama)
-- Naikkan gaji 10% untuk karyawan di dept Research
UPDATE EMPLOYEE
SET   Salary = Salary * 1.1  -- kanan: nilai LAMA, kiri: nilai BARU
WHERE Dno IN (
  SELECT Dnumber
  FROM   DEPARTMENT
  WHERE  Dname = 'Research'
);

💡 Assignment vs Reference: Dalam SET Salary = Salary * 1.1, Salary di kiri tanda = adalah kolom tujuan (nilai BARU), sedangkan Salary di kanan adalah nilai LAMA sebelum dimodifikasi.

Q1

Anda membuat tabel EMPLOYEE. Setelah itu Anda ingin menambah kolom PHONE VARCHAR(15) NOT NULL menggunakan ALTER TABLE. Apa yang terjadi?

A
Perintah berhasil, semua baris lama otomatis diisi dengan string kosong ''
B
Perintah berhasil, semua baris lama otomatis diisi NULL, constraint NOT NULL aktif untuk INSERT berikutnya
C
Perintah gagal/error karena NOT NULL tidak diizinkan saat ALTER TABLE ADD COLUMN pada tabel yang sudah berisi data
D
Perintah berhasil dan NOT NULL langsung berlaku untuk semua baris lama

📖 Metodologi Penyelesaian

Aturan ALTER TABLE ADD COLUMN: Ketika kolom baru ditambahkan ke tabel yang sudah berisi data (baris), semua baris yang sudah ada tidak memiliki nilai untuk kolom tersebut → DBMS mengisi dengan NULL.

Karena semua baris lama akan menjadi NULL, menambahkan constraint NOT NULL sekaligus tidak mungkin — itu berarti constraint langsung dilanggar oleh semua baris yang ada.

✅ Solusi: Tambah kolom tanpa NOT NULL → UPDATE semua baris untuk mengisi nilai → baru tambahkan constraint NOT NULL via ALTER TABLE.

Q2

Tabel EMPLOYEE memiliki kolom Dno INT NOT NULL DEFAULT 1 dengan FK ke DEPARTMENT. Kolom Dno di EMPLOYEE menggunakan ON DELETE SET DEFAULT. Jika department nomor 5 dihapus, apa yang terjadi pada karyawan di dept 5?

A
Karyawan di dept 5 ikut terhapus dari tabel EMPLOYEE
B
Nilai Dno karyawan di dept 5 berubah menjadi 1 (nilai DEFAULT)
C
Nilai Dno karyawan di dept 5 berubah menjadi NULL
D
DELETE gagal karena masih ada karyawan yang merujuk dept 5

📖 Metodologi Penyelesaian

Tabel aksi referential integrity:

• CASCADE → baris referencing ikut dihapus/diubah | • SET NULL → FK di-set NULL | • SET DEFAULT → FK di-set ke nilai DEFAULT | • RESTRICT → operasi gagal

✅ B benar: Karena aturannya ON DELETE SET DEFAULT dan DEFAULT Dno adalah 1, maka ketika dept 5 dihapus, semua karyawan dept 5 mendapat Dno = 1.

Q3

Perhatikan perintah berikut:
DELETE FROM EMPLOYEE;
Apa perbedaan utama dengan DROP TABLE EMPLOYEE;?

A
DELETE FROM menghapus semua baris data tetapi struktur tabel tetap ada; DROP TABLE menghapus data sekaligus definisi/struktur tabelnya
B
Keduanya identik — hasil akhirnya sama
C
DELETE FROM lebih berbahaya karena tidak bisa di-undo, sementara DROP TABLE bisa
D
DROP TABLE hanya menghapus data, struktur masih bisa dipakai untuk INSERT berikutnya

📖 Metodologi Penyelesaian

DELETE FROM tabel adalah operasi DML yang menghapus data (baris/tuples), bukan struktur. Tabel masih ada dan bisa menerima INSERT/SELECT baru. Ibaratnya: "mengosongkan isi lemari, lemarinya masih ada."

✅ DROP TABLE tabel adalah operasi DDL yang menghapus tabel + semua definisinya. Tabel tidak bisa lagi digunakan dalam query apapun. Ibaratnya: "merobohkan lemarinya sekalian."

Q4

Apa yang salah dari perintah UPDATE berikut untuk menaikkan gaji 10% bagi seluruh karyawan?
UPDATE EMPLOYEE SET Salary = 37000 * 1.1;

A
Perintah akan error karena tidak ada klausa WHERE
B
Semua karyawan akan mendapat gaji yang sama: 37000 * 1.1 = 40700, bukan naik 10% dari gaji masing-masing
C
Perintah benar dan valid untuk menaikkan gaji 10% semua karyawan
D
Perintah akan error karena Salary tidak boleh digunakan di kanan tanda =

📖 Metodologi Penyelesaian

Masalahnya: 37000 * 1.1 adalah nilai literal konstan. Semua karyawan, apapun gaji aslinya, akan di-set ke 40700. Ini bukan "naik 10% dari gaji masing-masing".

Yang benar: SET Salary = Salary * 1.1 — di sini Salary di kanan = merujuk pada nilai LAMA kolom Salary untuk setiap baris, lalu dikalikan 1.1 menjadi nilai BARU.

✅ Klausa WHERE tidak wajib; tanpa WHERE semua baris diperbarui (ini valid tapi berbahaya jika tidak disengaja).

Q5

Mengapa foreign key Super_ssn di tabel EMPLOYEE tidak bisa langsung dideklarasikan saat CREATE TABLE EMPLOYEE?

A
Karena Super_ssn bukan primary key, sehingga tidak bisa dijadikan FK
B
Karena tipe data CHAR(9) tidak bisa digunakan sebagai FK
C
Karena FK hanya bisa dideklarasikan menggunakan ALTER TABLE, bukan CREATE TABLE
D
Karena Super_ssn merujuk ke tabel yang sama (EMPLOYEE) yang belum selesai dibuat saat itu — self-referencing circular

📖 Metodologi Penyelesaian

Self-referencing FK: Super_ssn adalah FK yang merujuk ke kolom Ssn di tabel EMPLOYEE itu sendiri. Artinya seorang karyawan bisa punya "supervisor" yang juga seorang karyawan.

Masalah: Saat perintah CREATE TABLE EMPLOYEE dieksekusi, tabel EMPLOYEE belum ada/belum selesai terdefinisi. Jika langsung dideklarasikan FK REFERENCES EMPLOYEE(Ssn), DBMS belum "tahu" tabel tersebut.

✅ Solusi: Buat EMPLOYEE tanpa FK Self-ref dulu, lalu gunakan ALTER TABLE EMPLOYEE ADD FOREIGN KEY (Super_ssn) REFERENCES EMPLOYEE(Ssn).
