BAB 1

Pengenalan Basis Data

  • Database: Kumpulan data yang punya makna tersembunyi.
  • DBMS: Software pembuat, pengelola & pengguna DB.
  • DB System: Gabungan Database + Software DBMS.
  • Sifat Kunci: Self-describing (memiliki meta-data/katalog), Isolas Program-Data, Data Abstraction interaktif.
  • Aktor Balik Layar: DBA, DB Designer.
  • Aktor Pemakai: End Users, System Analyst, App Programmer.
BAB 2

Konsep & Arsitektur

  • Data Model: Abstrak mendefinisikan struktur + constraints. High (ER), Mid (Relational), Low (Fisik).
  • Three-Schema:
    • - Internal: Struktur fisik (B-tree, index).
    • - Conceptual: Logical structure untuk semua user.
    • - External: User View turunan aman.
  • Data Independence: Logical (Ubah Concept tanpa ngefek External) & Physical (Ubah fisik tanpa ngefek Concept).
  • Bahasa: DDL (Definisi structure), DML (Manipulasi Data).
BAB 3

Pemodelan ER (Dasar)

  • Entitas & Atribut: Objek (Entity). Tipe Atribut: Simple, Composite (Pecahan), Multi-valued (Opsi banyak), Derived (Kalkulasi).
  • Keys: Pembeda unik, simbolnya atribut bergaris bawah.
  • ⭐ Derajat Relasi: Unary/Rekursif (Entitas relasi ke dirinya sendiri, wajib beri Role Name), Binary (1:1, 1:N, M:N), Ternary (3 Entitas tergabung).
  • Participation:
    • - Total/Existence: Wajib (Garis ganda).
    • - Partial: Boleh tidak ikut (Garis tunggal).
  • Weak Entity: Ketergantungan ke Owner via Identifying Rel (BelahKetupat Ganda). Pakai Partial Key (Garis putus-putus).
BAB 4

Pemodelan EER (Lanjut)

  • Inheritance: Subclass menyebar warisan Atribut+Relasi dari Superclass induk (Simbol 'd'/'o' bulatan).
  • Disjointness:
    • - d Disjoint: Murni cuma bisa satu opsi kelas.
    • - o Overlap: Boleh ada 2 tipe status subclass sekaligus.
  • Completeness (Total/Partial):
    • - Wajib total (Garis ganda dari sup ke lingkar). Tiap object Induk wajib ada tipe anaknya.
  • Union/Category (U): Object subclass hasil persilangan bermacam-macam Tipe/Macam Superclass acak.
BAB 5

Model Relasional & Integritas

  • Komponen: Relation (Tabel), Tuple (Row/Baris), Atribut (Kolom/Garis).
  • Ketentuan Atribut: WAJIB Atomic/Tunggal tak boleh pecahan Array.
  • Kunci/Keys:
    • - Superkey: Combinasi apapun pencegah kembar.
    • - Candidate Key: Superkey terpendek / minimalis.
    • - Primary Key (PK): Pemenang Candidate (garis bawah).
  • Constraints Integritas: Entity Integrity (PK gak boleh = Null). Referential Integrity (Sebuah Foreign Key [FK] hrs Valid mengarah ke Tabel tujuan/NULL. Arahnya dr FK -> tabel ref).
BAB 6

Pemetaan Skema ke Tabel

  • Langkah Cepat Algoritma ER:
    • 1. Regular: Tabel tunggal. Weak: PK=PK Parent + PartialKey.
    • 2. Map 1:1: Sisipkan FK di sisi Total Participation.
    • 3. Map 1:N: Beri FK di sisi Entity banyak (N).
    • 4. Map M:N (WAJIB Ada Junction!): Buat tabel penengah, isinya FK dari entity A & B, lalu gabungkan sebagai Composite PK.
    • 5. Map Multivalued: Buat Tabel Khusus berisi (Nilai Itunya + FK Parent). Keduanya jadi gabungan PK.
    • 6. Map Rekursif (Unary 1:N): Tambahkan 1 Atribut FK di tabel itu sendiri yg menunjuk PK-nya (mis: Super_ssn menunjuk Ssn).
  • ⭐ Map EER (Penting!):
    Bila Disjoint (d): 1 Tabel induk dgn tipe khusus (Discriminator Col).
    Bila Overlap (o): 1 Tabel induk dgn banyak kolom Boolean Flag (IsTypeA, IsTypeB).
BAB 7

SQL - DDL (Data Def)

  • Konstruksi: CREATE SCHEMA, CREATE TABLE x (...).
  • Tipe Data Canggih: VARCHAR(n) dinamis asalkan di bwh n limit. CHAR(n) static buang tempat bila tak sampai length.
  • Contoh Table Config:
    Dno INT NOT NULL,
    PRIMARY KEY (Ssn),
    FOREIGN KEY (Dno) REFERENCES DEPT(Dnum)
  • Ref Integritas Eksekusi: ON DELETE / ON UPDATE punya Opsi: CASCADE (Ekoran ikut dihapus), SET NULL, SET DEFAULT.
BAB 8

Kueri Dasar & Agregat SQL

  • Dasar Join & String: WHERE E.Dno = D.Dnumber dan LIKE '%A_'.
  • ⭐ Subqueries/Nested: Pakai klausa IN, ANY, ALL, atau EXISTS untuk menghubungkan kueri utama dngn kueri sarang.
  • 🔥 Agregat (Pasti Keluar!): MAX(), MIN(), AVG(), SUM(), COUNT().
  • 🔥 GROUP BY: Memilah tabel untuk diagregasi per-grup (cth: Rata2 Gaji Tiap Dept).
    Syarat: Atribut mandiri di SELECT wajib ada di GROUP BY!
  • 🔥 HAVING: WHERE-nya para Agregat! Digunakan untuk memfilter hasil grup.
    Cth: HAVING COUNT(*) > 2. Urutan baca: WHERE -> Group By -> HAVING.
  • Set Ops: UNION, EXCEPT, INTERSECT (Wajib Kompatibel).
