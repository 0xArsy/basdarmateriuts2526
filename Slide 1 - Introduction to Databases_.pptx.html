Penyelaman Konsep Database

Apa itu Database?

Database adalah kumpulan data yang saling berhubungan (related data) — fakta-fakta yang diketahui yang dapat dicatat dan memiliki makna implisit. Anggap saja sebagai lemari arsip digital terstruktur di mana setiap bagian informasi saling terhubung dan memiliki tujuan tertentu.

Istilah Kunci
Database
Kumpulan data yang saling berhubungan yang diorganisasikan, dibangun untuk tujuan tertentu.
Meta-data
Data mengenai data — schema yang disimpan dalam katalog (nama kolom, tipe data, dll).
Query
Permintaan untuk mengambil data spesifik tanpa mengubahnya.
Transaction
Operasi read/write yang dieksekusi sebagai satu kesatuan utuh (atomic).
Kesalahan Umum (Pitfall)

Tidak semua kumpulan file adalah database! Sebuah folder berisi spreadsheet acak tidak memiliki tiga properti esensial: representasi miniworld, keterpaduan logis (logical coherence), dan tujuan spesifik. Sekadar menyimpan data ≠ memiliki database.

Model Mental

Bayangkan sebuah sistem catatan mahasiswa di universitas. "Miniworld"-nya adalah universitas tersebut. Data (nama, mata kuliah, nilai) secara logis koheren dan dibangun khusus untuk mengelola catatan akademik — bukan hanya dibuang begitu saja ke dalam file.

Miniworld (UoD)

Merepresentasikan suatu aspek di dunia nyata. Perubahan yang terjadi pada miniworld tercermin dalam database.

Keterpaduan Logis (Logically Coherent)

Data memiliki makna dan relasi inheren — bukan sekumpulan informasi acak.

Tujuan Spesifik

Dibangun untuk kelompok pengguna yang dimaksudkan dan aplikasi yang telah dirancang sebelumnya — tidak dibuat secara kebetulan.

Istilah Kunci: Universe of Discourse (UoD)

Bagian dari dunia nyata yang dimodelkan oleh database Anda. UoD database perpustakaan mencakup buku, anggota, peminjaman — tapi bukan cuaca di luar. Apa pun di luar UoD tidak boleh muncul dalam schema Anda.

Kesalahan Umum (Pitfall)

Membingungkan antara data dengan informasi. Data adalah fakta mentah. Informasi adalah data yang telah diproses dan diberi konteks. Database menyimpan data; query mengubahnya menjadi informasi.

Apa itu DBMS?

Database Management System adalah kumpulan program yang memungkinkan pengguna untuk membuat dan memelihara database. Bertindak sebagai perantara antara user/aplikasi dengan data tersimpan yang sebenarnya.

PostgreSQL
MySQL
MongoDB
SQL Server
Redis
SQLite
Oracle
Cassandra
Elasticsearch
Model Mental

DBMS ibarat sistem manajemen perpustakaan. Buku = data. Katalog = meta-data. Pustakawan = DBMS — menegakkan aturan siapa yang meminjam apa, kapan, dan berapa banyak sekaligus.

Fungsi Kunci DBMS
→ Define (Mendefinisikan): Menentukan tipe data, struktur, constraints
→ Construct (Membangun): Menyimpan data ke medium penyimpanan
→ Manipulate (Memanipulasi): Query, update, menghasilkan reports
→ Share (Berbagi): Mengizinkan akses untuk berbagai pengguna/aplikasi secara bersamaan

Ada dua kelompok yang berinteraksi dengan database system: mereka yang berada di lokasi (Actors on the scene) dan mereka yang bekerja di balik layar (Workers behind the scene).

Actors on the Scene (Aktor yang menggunakan database)

Database Administrator (DBA)
Memberikan otorisasi akses, memonitor penggunaan, dan mengurus resource software/hardware.
Database Designers
Mengidentifikasi data yang akan disimpan; memilih struktur yang tepat untuk merepresentasikannya.
End Users (Pengguna Akhir)
Naive / Casual / Sophisticated / Standalone — mereka yang melakukan query atau update pada data sehari-hari.
System Analysts & App Programmers
Menentukan requirement dan mengimplementasikan program yang akan digunakan ke DBMS.

Workers Behind the Scene (Pekerja di Balik Layar)

DBMS Designers & Implementers
Mendesain dan membuat software DBMS itu sendiri (contoh: tim yang membuat PostgreSQL).
Tools Developers
Membuat tools opsional untuk desain DB, pemantauan performa, atau import/export data.
Operators & Maintenance
Menjalankan dan merawat lingkungan hardware/software tempat database system berjalan.
Perbedaan Kunci

Actors menggunakan database. Workers membangun dan merawat infrastrukturnya. DBA berada di perbatasan — mereka menggunakan tools DBMS untuk mengelola sistem bagi kepentingan user.

GUNAKAN Database
  • Data bisnis besar yang butuh sering di-update
  • Catatan data yang sifatnya mirip/homogen dalam skala besar
  • Data harus tersimpan / relevan dalam jangka waktu lama
  • Butuh akses secara bersamaan (multi-user)
TIDAK Butuh
  • Aplikasi sederhana, statis, tidak pernah berubah
  • Terdapat batasan real-time yang ketat
  • Sistem tertanam (embedded) atau storage terbatas
  • Single user, tanpa perlu concurrency
TIDAK BISA Digunakan
  • Kompleksitas data melebihi batas modeling dari DBMS
  • User butuh operasi khusus yang tidak dapat ditangani DBMS
Model Mental

Menggunakan DBMS untuk 50 baris catatan belanja pribadi sama seperti memakai truk kargo industri hanya untuk membawa tas belanjaan — terlalu berlebihan (overhead tinggi) namun sedikit benefitnya. Sesuaikan tools dengan skala masalahnya.

Algoritma & Analisis Logika Secara Berurutan (Walkthrough)

Operasi dasar SQL pada komponen University Database yang diambil dari slide material dengan penjelasan langkah demi langkah (walkthrough) logika operasionalnya.

Definisi Schema — (DDL)SQL
-- Create STUDENT table (meta-data / structure definition)
CREATE TABLE STUDENT (
  Name           CHAR(30),
  Student_number CHAR(4),
  Class          INTEGER(1),
  Major          Major_type
);
CREATE TABLE COURSE (
  Course_name    CHAR(10),
  Course_number  CHAR(8),
  Credit_hours   INTEGER,
  Department     CHAR(4)
);
CREATE TABLE GRADE_REPORT (
  Student_number     CHAR(4),
  Section_identifier INTEGER,
  Grade              CHAR(1)
);
Contoh Queries & Updates (DML)SQL
-- Q1: List nama mahasiswa di matkul 'Database' pada semester 'Fall' 2008 & nilai mahasiswa
SELECT s.Name, gr.Grade
FROM   STUDENT      s
JOIN   GRADE_REPORT gr  ON s.Student_number      = gr.Student_number
JOIN   SECTION      sec ON gr.Section_identifier = sec.Section_identifier
JOIN   COURSE       c   ON sec.Course_number     = c.Course_number
WHERE  c.Course_name = 'Database'
  AND  sec.Semester  = 'Fall'
  AND  sec.Year      = 08;

-- Q2: Prerequisites pada mata kuliah 'Database'
SELECT p.Prerequisite_number
FROM   PREREQUISITE p
JOIN   COURSE       c ON p.Course_number = c.Course_number
WHERE  c.Course_name = 'Database';

-- U1: Update class untuk mahasiswa Smith menjadi tingkat dua ('sophomore') / '2'
UPDATE STUDENT
SET    Class = 2
WHERE  Name  = 'Smith';
